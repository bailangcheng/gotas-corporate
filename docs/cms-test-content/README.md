# microCMS 表示テストキット

`magazine` と `news` の記事ページが、**画像が複数枚穿插された複雑な本文**と
**青いハイパーリンク**を正しく表示できるかを検証するための一式。

## 中身

| ファイル | 用途 |
| --- | --- |
| `fields.md` | 各フィールド(タイトル/スラッグ/抜粋/タグ等)のコピペ用の値 |
| `magazine-article-body.html` | マガジン本文。画像4枚＋青リンク3本＋見出し/箇条書き/引用 |
| `news-article-body.html` | お知らせ本文。画像1枚＋青リンク2本 |
| `../../scripts/seed-cms.mjs` | API で記事を自動投入するスクリプト（投放） |

> ## ⚠️ 重要: 画像は API では入らない（microCMS の仕様）
>
> microCMS の richEditor は **Write API 経由でも HTML をサニタイズ**し、
> **管理画像(`images.microcms-assets.io`)以外の `<img>` を除去**する。
> 相対パスや外部URLの画像は保存時に消える（検証済み）。
> 一方、**見出し・箇条書き・太字・引用・青リンクはそのまま通る**。
>
> → **本文テキスト＋リンク＝API（方法B）で投入可。画像だけは管理画面の
> エディタ（画像ボタン）でアップロードする**（= microcms-assets URL になり表示される）。
> 描画コード側は `<img>` 対応済み（magazine の `[&_img]` スタイル）なので、
> 管理画像を入れれば何枚でも正しく表示される。
>
> `.html` ファイルは「最終的にどう組むか」の設計図兼プレビュー用。
> ブラウザで開けば画像入りの完成イメージを確認できる。

---

## 投入方法は2通り

### 方法A — 管理画面で手で作る（一番リアル、推奨）

実際の編集者の運用そのものをテストできる。

1. microCMS にログイン → `マガジン` → **追加**
2. `fields.md` の表のとおり タイトル / スラッグ / カテゴリ / タグ / 抜粋 を入力
3. 本文(`body`)を組む:
   - 段落のテキストを入力
   - 段落の間で、ツールバーの **画像ボタン**から写真を挿入（複数回）→ 画像が
     `images.microcms-assets.io` にアップロードされる
   - リンクにしたい文字を選択 → **リンクボタン**でURLを設定 → 青い下線リンクになる
4. 右上 **公開**
5. `/magazine/gosei-kaisendon-open` を開いて確認（下のチェックリスト）

> 写真は手持ちのどんな画像でもOK。枚数も自由（何枚でも表示される）。

### 方法B — スクリプトで自動投入する（投放 / 速い・正確）

`magazine-article-body.html` の本文HTMLをそのまま API で送り込む。画面操作ゼロ。

```sh
# 書き込み権限のある Management API キーが必要
MICROCMS_SERVICE_DOMAIN=go-tas \
MICROCMS_MANAGEMENT_API_KEY=＜書き込み可能なキー＞ \
node scripts/seed-cms.mjs
```

- 先に `node scripts/seed-cms.mjs --dry-run` で送信内容だけ確認できる
- 本文中の画像は外部URL/相対パスの生 `<img>` なので、そのまま表示される
- カバー画像だけは API から URL 指定できない（メディア事前アップロードが必要）。
  必要なら管理画面で1枚だけ足す。
- 投入後、下書き状態なら管理画面で **公開** に変更する

## 確認チェックリスト

`/magazine/gosei-kaisendon-open`（方法A/B 共通）で:

- [ ] 本文中の画像が **4枚すべて** 表示される
- [ ] 画像が角丸・横幅いっぱいで崩れず並ぶ
- [ ] 写真キャプション（figcaption）が画像の下に中央寄せで出る
- [ ] Googleマップ / 公式サイト / Instagram の3本が **青い下線リンク**で表示される
- [ ] リンクをクリックすると正しく遷移する
- [ ] 見出し(h2/h3)・箇条書き・引用(blockquote)・太字が整って見える
- [ ] スマホ幅でも画像が横にはみ出さない

`/news/<slug>`（news エンドポイント作成後）で:

- [ ] 本文中の画像が表示される（※news側の画像スタイルは今回のパッチで対応済み）
- [ ] 青リンクが表示・遷移する
